Principal Movement In Arbitrum Positions

The dashboard tracks two ledgers at once. Quantity follows the token actually held. Principal follows the economic family when the exposure is clearly one family; otherwise it stays on the active wrapper until accounting can split it.

Quantity ledger Principal ledger Mixed LP rule Aave debt rule

The Core Rule

1

Quantity is literal. If the wallet holds aArbwstETH, raw snapshots show quantity on aArbwstETH, not ETH.

2

Principal is economic. If every terminal component resolves to one family, principal is redirected to that family, like WETH and wstETH ending on ETH.

3

Mixed LPs stay active. A USDC/USDT LP keeps principal on the LP in raw snapshots. Accounting expands it into USDC and USDT for dashboard reporting.

Raw Versus Dashboard

Raw snapshots preserve position mechanics. Accounting converts those mechanics into base exposure rows for P/L.

Layer What it stores Example Why it matters
Raw snapshots Actual token quantity and the current principal holder. aArbwstETH quantity, ETH principal. Lets transfers and unwraps avoid creating artificial realized P/L.
Accounting source rows Expanded base exposure per source. Aave -> ETH market value with ETH principal elsewhere. Combines wrappers, protocols, and direct tokens into comparable base assets.
Dashboard asset rows Base selection totals and P/L. ETH ProfitLossEUR = MarketValueEUR - PrincipalInvestedEUR. Shows economic result instead of wrapper mechanics.
Deposits are transformations.

Moving into a wrapper or Aave supply token should not realize a gain or loss by itself.

Prices still move P/L.

Once principal is stable, daily P/L changes should mainly come from market value changes.

Mixed baskets need weights.

When there is no single base family, accounting uses component values to allocate principal.

Debugging Shortcut

If P/L jumps on a wrapper deposit or withdrawal day, compare the base asset's MarketValueEUR delta with its PrincipalInvestedEUR delta. A price move changes market value. A synthetic cliff usually means principal appeared, disappeared, or moved to an intermediate wrapper at the wrong layer.
